# 工作原理
HxASM 的运行流程遵循经典的解释器架构：

- Scanner (扫描器)
从标准输入读取源码
处理动态内存分配，支持任意长度的单行指令输入。

- Lexer (词法分析器)
将原始字符串解析为 Token 序列（tokens）
识别关键字（如 push, add）和数字，并将其转化为内部枚举类型（如 OP_PUSH, NUMBER）。

- VM (虚拟机)
核心是一个长度为8的操作数栈 (OpStack)
解释器遍历 Token 序列，根据指令操纵栈顶指针 (top) 和栈内数据。
计算逻辑采用后缀表达式（逆波兰表示法）的风格。

## 语法(巴科斯范式)
```text
<数字>::= [0-9]+
<压栈>::= <"push"> <数字>
<出栈>::= <"pop">
<次栈顶加栈顶并将栈顶指针回移>::= <"add">
<次栈顶减栈顶并将栈顶指针回移>::= <"sub">
<次栈顶乘栈顶并将栈顶指针回移>::= <"mul">
<次栈顶除以栈顶并将栈顶指针回移>::= <"div">
<输出栈项的值但不移动栈顶指针>::= <"out">
```
## 限制与特性
- 栈溢出保护：操作数栈的最大深度被硬编码为 8。如果入栈超过 8 层，或者在空栈时弹出，VM 会报错;
- 整数运算：目前仅支持整数（long 类型）运算;
- 本地化：程序会自动尝试设置中文环境（zh_CN.UTF-8）;
